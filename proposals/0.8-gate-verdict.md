# 0.8 — the structured gate verdict (`--gate-json`)

Status: **all four engines implement it; the floor has risen to `0.8`.** candor-java led (as the reference),
then candor-scan, candor-ts and candor-swift; their agreement is pinned by the conformance gate-verdict
differential (PART 12). First version to ride the **ladder** (SPEC.md §"Versioning policy"): a minor rung led
by the reference engine, then implemented across the rest in turn — not stamped across all four at once.

## Motivation

candor enforces a §6.2 policy at scan time and signals the result two ways: a process **exit code** (1 on a
violation) and human **`AS-EFF` console lines**. Neither is machine-readable, so a consumer that wants to
*do something* with the violations — annotate a pull request, write SARIF for the Code-scanning tab, feed an
agent — has to parse console prose. That parsing is exactly the brittleness candor tells its users to avoid.

0.8 adds a **structured** re-emission of the verdict the engine already computed.

## Shape

```text
gate  { "spec": "<version>", "ok": bool, "violations": [ { "rule", "fn", "effects":[Effect…], "detail"? } ] }
```

- `ok` — the CI verdict: `true` ⇔ the run gate-passes. Advisory-only findings (e.g. `AS-EFF-007`) MAY
  appear in `violations` but MUST NOT clear `ok`.
- `rule` — the `AS-EFF-00x` code (§6) that fired.
- `fn` — the function it fired on (language-natural name, as elsewhere in §3.1).
- `effects` — the specific effect(s) the violation concerns: the DENIED set (what the entity does ∩ what the
  rule forbids), e.g. `["Fs"]` for a `{Clock, Fs}` fn under `deny Fs`. `[]` for a code with no single effect
  (`AS-EFF-009` layer-flow, `AS-EFF-003` unresolved). This is the piece a consumer CANNOT reconstruct from
  the report — the report's per-fn `direct` set is a superset of the denied effects.
- `detail` — OPTIONAL human message (the engine's own diagnostic text).

Emitted via `--gate-json <file>` (§3.3). Written whenever a gate is active; with none configured it writes
the clean verdict `{ ok: true, violations: [] }`. Does not change the exit code.

## Invariants

- **Agreement with the exit code.** The verdict is a re-emission of the gate the engine already ran, from
  the *same* check — a non-empty gate-failing `violations` ⟺ exit 1. A consumer can never see a verdict that
  disagrees with the gate. (candor-java achieves this by capturing at its single diagnostic sink, so every
  `AS-EFF` code is recorded once, with no separate re-derivation.)
- **Additive / off by default.** No report-envelope change; producing the verdict must not alter the §2
  report or the console output. `spec` stays `0.7` for an engine that hasn't added `--gate-json`.
- **Locations not duplicated.** The verdict carries `fn`, not source spans — a consumer joins each `fn` to
  its `loc` in the §2 report. (This is what the PR-native SARIF reporter, `candor/integrations/github`, does:
  `fn` → `loc` → a SARIF `physicalLocation`, and `effects` → the effect a `path` query traces into a SARIF
  `codeFlow`.) `effects` IS in the verdict precisely because the report can't supply the denied subset.

## Conformance

The cross-engine differential feeds every engine that declares `0.8` the same code + policy, runs the
gating command with `--gate-json`, and pins **`ok` and the `{rule, fn, effects}` set**. `detail` is engine-natural
prose and is NOT pinned (like the function-name *value* in §3.1). Engines still on `0.7` are excluded from
this differential — that is the ladder: agreement is proven across the engines that have reached the rung,
and the floor rises to `0.8` when the last engine lands `--gate-json`.

## Rollout

1. candor-java — `--gate-json`, captured at the `Candor.diag` sink. **Done** (declares `0.8`).
2. candor-scan / candor-ts / candor-swift — `--gate-json` from each engine's policy checker (structured
   violations `{rule, fn, effects, detail}`). **Done** — all declare `0.8`.
3. Conformance gate-verdict differential (PART 12) — the same static deny-Fs fixture in all four languages;
   every engine's verdict must agree (ok:false · AS-EFF-006 `save` · {Fs}). **Done** — floor risen to `0.8`.
